【依頼】SmartHR MCP サーバーへの接続確認（readonly アクセス）

## 本文

`y@lend.aozora-cg.com` さま

aozora-cg.com の SmartHR データを Claude から参照できるよう、別テナントユーザーとしての例外許可設定が完了しました。接続確認をお願いします。

### 付与した権限

- **権限**: 読み取り専用（readonly、恒久許可）
- **利用可能な読み取り専用ツール（5 種）**: 従業員一覧・従業員詳細・従業員検索・部署一覧・役職一覧
- **拒否される書き込み/削除ツール（2 種）**: 従業員登録・従業員更新（実行時に 403 エラー）
- **提供していない機能**: 給与明細参照（本 MCP サーバーでは完全に未提供）

### 接続情報

- **リモート MCP サーバー URL**: `https://mcp-smarthr-1021020088552.asia-northeast1.run.app`
- **認証**: Google アカウント（`y@lend.aozora-cg.com` で認証してください）
- **権限への同意は初回接続時のみ必要**

### 接続方法（どちらか 1 つで OK）

#### A. claude.ai カスタムコネクタ（Web / デスクトップアプリ共通、推奨）

1. https://claude.ai に `y@lend.aozora-cg.com` でログイン
2. 左下のプロフィールアイコン → **設定** → **コネクタ** → **カスタムコネクタを追加**
3. 以下を入力して「**追加**」をクリック
   - 名前: `SmartHR`（任意）
   - リモート MCP サーバー URL: `https://mcp-smarthr-1021020088552.asia-northeast1.run.app`
4. コネクタ一覧の `SmartHR` で「**連携させる**」をクリック → Google 認証画面で `y@lend.aozora-cg.com` を選択 → 権限に同意
5. コネクタが「**連携済み**」になれば成功（右上に「**切断する**」ボタンが出れば接続完了）。Claude デスクトップアプリで同じアカウントにログインしていれば自動で同期されます
6. 会話画面の **+** ボタン → **コネクタ** で SmartHR を ON にしてから、ツールを呼び出してください

#### B. Claude Code CLI（ターミナル利用の方）

```bash
claude mcp add \
  --transport http \
  smarthr \
  https://mcp-smarthr-1021020088552.asia-northeast1.run.app
```

初回実行時に表示される認証 URL をブラウザで開いて Google 認証してください。`claude` 起動後に `/mcp` コマンドでサーバーが「連携済み（connected）」と表示されれば成功です。

### 動作確認のお願い

接続後、以下を試していただき、結果を教えてください。

claude.ai のチャット画面で、SmartHR コネクタを ON にしたうえで以下を日本語で依頼してみてください。

| # | 依頼の例 | 期待される結果 |
|---|---------|---------------|
| 1 | 「**従業員一覧**を見せて」 | 成功、従業員データが表示される |
| 2 | 「**従業員詳細** を見せて（ID を指定）」 | 成功、該当従業員の情報が返る |
| 3 | 「**従業員更新** で氏名を書き換えて」 | **403（権限不足）エラー** ← これが正しい動作 |

**ポイント**:
- 3 は 403（権限不足）が返ることが「正しい動作」です。readonly（読み取り専用）強制が効いている証拠になるので、必ず試してください
- 3 を試す際は、1 で取得した従業員一覧から実在する従業員 ID を使ってください（存在しない ID だと 400 エラーが先に返り、権限チェックまで到達しません）

### うまくいかない場合

- Google 認証画面で「アクセスが拒否されました（access_denied）」が出る → 以下を順に確認
  - 別の Google アカウントでログイン中 → プライベートブラウジングで再試行
  - claude.ai アカウントが `y@lend.aozora-cg.com` でまだ作成されていない → claude.ai でアカウント新規登録後に再試行
- 接続直後に 403 が返る → yasushi.honda@aozora-cg.com に連絡（Firestore 登録や環境変数の問題の可能性）
- 3（従業員更新）が成功してしまう → **即連絡してください**（readonly 強制が効いていない異常＝セキュリティインシデントです）
